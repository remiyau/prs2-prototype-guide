Getting Started with GitLab

What you need

  • A computer running macOS
  • A GitLab account on sgts.gitlab-dedicated.com
  • Git installed (Step 1 below)
  • Visual Studio Code — download from code.visualstudio.com
1

Install Git

macOS

  1. Open Terminal (press Cmd + Space, type "Terminal", press Enter)
  2. Type git --version and press Enter
  3. If Git is not installed, macOS will ask you to install the Xcode Command Line Tools. Click Install and wait for it to finish.
2

Configure Git with your name and email

This tells Git who you are. Run these commands in Terminal (Mac) or Git Bash (Windows), replacing the placeholders with your actual name and email:

git config --global user.name "Your Full Name" git config --global user.email "your.email@example.com"
Important: Use the same email address linked to your GitLab account.
3

Create an SSH Key

An SSH key lets your computer talk to GitLab securely — no password needed every time you push or pull.

Generate the key:

ssh-keygen -t ed25519 -C "your.email@example.com"
  1. Press Enter to accept the default file location
  2. When asked for a passphrase, press Enter to skip (or type one for extra security)

Add the key to the SSH agent:

ssh-add ~/.ssh/id_ed25519

Copy your public key to clipboard:

macOS

tr -d '\n' < ~/.ssh/id_ed25519.pub | pbcopy

Windows (Git Bash)

cat ~/.ssh/id_ed25519.pub | clip

Add the key to GitLab:

  1. Go to sgts.gitlab-dedicated.com and sign in
  2. Click your avatar (top right) → Edit profile
  3. In the left sidebar, select Access → SSH keys
  4. Click Add new key
  5. Paste the key into the Key field. It should look something like this:
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... your.email@example.com
  1. Give it a title (e.g. "Work Laptop 2025")
  2. Click Add key

Test the connection:

ssh -T git@sgts.gitlab-dedicated.com
Success: You should see a welcome message with your GitLab username.
4

Configure your signing key

This repository requires commits to be signed with your SSH key. Run this command:

git config --global user.signingkey ~/.ssh/id_ed25519.pub
Why? Without this, you'll get an error when trying to commit: fatal: either user.signingkey or gpg.ssh.defaultKeyCommand needs to be configured
5

Clone the repository

Cloning downloads a copy of the project onto your computer.

  1. Open Terminal (Mac) or Git Bash (Windows)
  2. Navigate to your Desktop:
cd ~/Desktop
  1. Clone the repo:
git clone git@sgts.gitlab-dedicated.com:wog/gvt/pic/prs/prs2-product-requirements.git
  1. Open the folder in VS Code:
code prs2-product-requirements
Done! You now have the full project on your computer. A folder called prs2-product-requirements appears on your Desktop.
6

Start Claude Code in VS Code

Claude Code is an AI assistant that runs in your terminal. It can help you create branches, edit files, and raise merge requests — all through plain English.

  1. In VS Code, open a new terminal: Ctrl + ` (Mac: Cmd + `) — or go to Terminal → New Terminal from the top menu
  2. Type the following and press Enter:
claude
That's it! Claude is now running inside your project. You can ask it to create a branch, make changes, or walk you through anything in this guide.
7

Create a new branch

Always work on a branch — never directly on main. Branches keep your changes isolated until they are reviewed.

  1. In VS Code, look at the bottom-left corner — it shows the current branch name (probably main)
  2. Click it. A dropdown appears at the top of the screen.
  3. Select Create new branch...
  4. Type a short name using hyphens, no spaces. For example:
update-ahpc-requirements
  1. Press Enter. VS Code switches to your new branch automatically.
Never commit directly to main. If you accidentally do, don't push — ask a team member for help.
