How to Create a PRS2.0 Interactive Prototype

A guide for designers to build interactive HTML prototypes using Claude Code — no coding experience required.

What you need before starting

  • Claude Code installed on your Mac (watch the video below if you haven't set it up)
  • Access to the prs2-product-requirements repository (cloned to your Desktop) — follow this guide to set up GitLab
  • Your Figma designs ready (you'll share screenshots with Claude)
  • Know which feature/module you're prototyping (e.g. CPE, Professionals, Registration)
▶

Watch: How to install Claude Code

If you haven't installed Claude Code yet, watch this video first. It walks through the full setup process.
Tip: The video starts at the installation steps. Once you have Claude Code running in your Terminal, come back here and continue with Step 1.
1

Open Claude Code in the project folder

  1. Open Visual Studio Code
  2. Go to File → Open Folder and select the prs2-product-requirements folder on your Desktop
  3. Open the built-in terminal: press Ctrl + ` (backtick key, above Tab) — or go to Terminal → New Terminal from the top menu
  4. In the terminal that appears at the bottom of VS Code, type:
claude

This starts Claude Code inside the project folder. It already knows about the PRDs, component library, and design tokens.

Tip: The VS Code terminal automatically opens in the project folder, so you don't need to cd anywhere. Just type claude and you're ready to go.
2

Tell Claude what you're building

Start by clearly describing what prototype you want. Be specific about:

  • Which portal (SPB Portal, HCP Portal, Public Portal, Corporate Portal)
  • Which feature/page (e.g. "Healthcare Institutions dashboard and profile page")
  • That you want a standalone HTML/CSS/JS prototype (no build tools)
Create a standalone HTML/CSS/JS interactive prototype of the SPB Portal Healthcare Institutions feature. It should have a dashboard page with a data table and a profile page with tabs. Make it so I can just open the HTML file in a browser — no build tools needed.
3

Share your Figma designs with Claude

Claude needs to see exactly what the prototype should look like. The fastest way is to export all your screens at once into a folder, then point Claude to it.

Option A: Export to a folder (recommended for multiple screens)

  1. In Figma, select all the frames you want prototyped (hold Cmd and click each frame)
  2. In the right panel under Export, choose PNG at 2x resolution
  3. Click Export selection — this exports all screens as a batch
  4. Save the exported images into a folder in your repo, e.g. Vibe-Coded-Apps/figma-screens/
  5. In Claude Code, tell it where to find the designs:
Look at all the images in Vibe-Coded-Apps/figma-screens/ and use them as reference for the prototype screens.
Why 2x? Exporting at 2x gives Claude crisp, high-resolution references — it can read text, see spacing, and match colours far more accurately than a regular screenshot.

Option B: Copy and paste directly

If you only need to share a few screens, you can do it one at a time:

  1. In Figma, select the frame
  2. Press Cmd + Shift + C to copy it as an image
  3. In Claude Code, paste with Cmd + V
Pro tip: Name your Figma frames clearly before exporting (e.g. "Dashboard", "Profile - Details Tab", "Create Modal"). The exported filenames will match your frame names, making it easier for Claude to understand which screen is which.
Important: Share ALL relevant screens — dashboards, profile pages, tab states, modals, and empty states. The more Claude sees, the more accurate the prototype will be.
4

Point Claude to the component library

Our project has an existing component library with exact colors, spacing, and styles. Tell Claude to use it so the prototype matches our design system perfectly.

Reference the component library code at Vibe-Coded-Apps/PRS2-Prototype/frontend/src/components/ for all UI — use its existing components, tokens, and styles exactly.

This makes Claude use the correct:

  • Colors (primary blue #286af8, text colors, borders)
  • Spacing and sizing (sidebar width, topbar height, row heights)
  • Component patterns (tables, badges, tabs, cards, buttons)
  • Typography (Inter font, weights, sizes)
Where the library lives: Vibe-Coded-Apps/PRS2-Prototype/frontend/src/components/
5

Export design tokens from Figma

The PRS Shadcn design library in Figma contains all the exact colour values, spacing, typography, and responsive breakpoints as JSON tokens. Exporting these gives Claude precise values instead of guessing from screenshots.

How to export:

  1. Open the PRS Shadcn design library file in Figma
  2. Click the Variables icon in the left sidebar (the circle icon)
  3. In the Collections list on the left, right-click on a collection (e.g. "1. TailwindCSS") → select Export
  4. Repeat for each collection (Theme, Mode, Desktop, Mobile, PRS styles)
  5. Save all exported JSON files into Vibe-Coded-Apps/figma-tokens/ in your project folder

Then tell Claude to use them:

Use the design tokens in Vibe-Coded-Apps/figma-tokens/ for exact colours, spacing, and typography.
Why? These tokens contain the exact values from the design system — Claude can match colours, font sizes, and spacing precisely without relying on what it sees in screenshots.
Pro tip: If the styling looks off after using the design tokens (e.g. wrong colours or typography), ask Claude to refer back to the Figma screenshots as the visual reference instead.
6

Point Claude to the relevant PRD

The PRDs contain the business logic — which columns to show, which tabs exist, what actions are available. Share the relevant one so the prototype is functionally accurate, not just visually correct.

Use the requirements from Requirements/Portals/SPB Portal/Features/Schools/ for the data model — columns, tabs, fields, and actions should match what's specified there.

Key PRD locations by portal:

  • Requirements/Portals/SPB Portal/ — SPB staff-facing portal
  • Requirements/Portals/HCP Portal/ — Healthcare professional portal
  • Requirements/Portals/Public Portal/ — Public-facing portal
  • Requirements/Portals/Corporate Portal/ — Corporate entity portal
  • Requirements/Processes/Boards/ — Process definitions (statuses, flows)
  • Requirements/Processes/Building Blocks/Forms/ — Form field definitions
7

Review and iterate

Claude will generate the prototype files. Once it's done:

  1. Open the generated index.html in your browser (double-click it)
  2. Compare it against your Figma designs
  3. Tell Claude what's wrong — be specific!

Good feedback examples:

The sidebar is too narrow — it should be 248px wide like in the design.
The table row height is wrong. Look at the screenshot again — rows should be taller with more padding.
The badge colors are wrong. Active should be green background, not a pill shape — it should be a rounded rectangle like in the component library.

You can also drop a new screenshot showing what's wrong vs. what you expected.

8

Where to save your prototype

All prototypes go in the Vibe-Coded-Apps/ folder. Tell Claude where to put it:

Save the prototype in Vibe-Coded-Apps/HCI-Prototype/ with index.html, profile.html, styles.css, and app.js.

The standard structure for a prototype:

  • Vibe-Coded-Apps/[YourFeature]-Prototype/
  • index.html — Main page (dashboard/list view)
  • profile.html — Detail/profile page (if needed)
  • styles.css — All styles in one file
  • app.js — All interactivity in one file

Quick Reference

The magic prompt (copy this as a starting point)

Here's a template you can paste into Claude Code and modify for your feature:

# Paste this into Claude Code and edit the parts in [brackets]: Create a standalone HTML/CSS/JS interactive prototype of the [SPB Portal / HCP Portal / Public Portal] [Feature Name] feature. Requirements: - Reference the component library at Vibe-Coded-Apps/PRS2-Prototype/frontend/src/components/ for all UI tokens and styles - Use requirements from [Requirements/Portals/SPB Portal/...] for data model and business logic - Standalone HTML that opens in a browser — no build tools - Save it to [Vibe-Coded-Apps/MyFeature-Prototype/] Pages needed: - [Dashboard with data table, search, filters, pagination] - [Profile page with tabs: Details, Permissions, etc.] Then I'll share Figma screenshots for the visual design.

Common things to ask Claude for

  • Add mock data — "Add 20 realistic rows with Singapore institution names and UENs"
  • Add interactivity — "Make the tabs switch content when clicked"
  • Add pagination — "Add pagination showing 10 rows per page"
  • Add search — "Add a search bar that filters the table by name"
  • Fix styling — "The badge color is wrong" + drop a screenshot
  • Add a new page — "Add a create form modal that appears when I click Add"
  • Match Figma exactly — drop a screenshot and say "make it look exactly like this"

Things to avoid

  • Don't edit the generated code manually — ask Claude to make changes instead
  • Don't delete the styles.css or app.js files — the HTML needs them to work
  • Don't be vague — "make it better" doesn't help; say exactly what's wrong
  • Don't forget the component library reference — without it, Claude will guess the colours and styles
